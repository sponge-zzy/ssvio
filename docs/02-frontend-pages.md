# 前端页面设计初稿

## 1. 前端目标

第一阶段前端目标是让初学者理解视频网站的页面结构、路由跳转、组件拆分、数据展示方式，以及基于 `localStorage` 的最小用户侧交互。

页面不连接真实后端，全部从 mock 数据读取影片。播放页使用 HTML5 `video` 播放测试视频地址。观看历史、收藏、评分、评论和弹幕只保存到浏览器 `localStorage`。

## 2. 第一阶段页面清单

### 首页

用途：

- 作为用户进入网站后的第一个页面。
- 展示推荐影片、最新影片、电影、电视剧、动漫等内容区域。

基础内容：

- 顶部导航栏
- 大横幅推荐影片
- 影片横向列表
- 分类入口
- 影片卡片

点击行为：

- 点击影片卡片进入影片详情页。
- 点击分类入口进入分类页。
- 点击搜索入口进入搜索页。

完成标准：

- 能展示 mock 影片列表。
- 至少有一个主推荐影片。
- 影片卡片能跳转到详情页。

### 分类页

用途：

- 展示和筛选影片。
- 第一阶段支持按分类、地区、年份筛选 mock 视频数据。

基础内容：

- 分类标题
- 分类筛选
- 地区筛选
- 年份筛选
- 影片网格

点击行为：

- 点击影片卡片进入影片详情页。

完成标准：

- 能展示分类页面基础 UI。
- 能按 mock 数据中的分类、地区、年份筛选影片。
- 筛选结果为空时有空状态提示。

### 搜索页

用途：

- 提供搜索入口和结果展示区域。
- 第一阶段实现基于 mock 数据的前端本地搜索。

基础内容：

- 搜索输入框
- 搜索按钮
- 搜索结果区域
- 默认推荐结果或全部影片

点击行为：

- 点击影片卡片进入详情页。

完成标准：

- 有搜索输入框。
- 有结果展示区域。
- 输入关键词后，结果可以根据影片标题、简介、分类、地区、年份和标签变化。
- 搜索结果为空时有简单提示。

### 影片详情页

用途：

- 展示单个影片的详细信息。
- 提供进入播放页的入口。

基础内容：

- 背景横幅图
- 封面图
- 影片标题
- 年份、地区、分类、题材
- 简介
- 播放按钮
- 收藏按钮
- 评分区域
- 评论输入框和评论列表

点击行为：

- 点击播放按钮进入播放页。
- 点击收藏按钮可以收藏或取消收藏。
- 点击评分按钮可以保存本地评分。
- 提交评论后可以在当前影片评论列表中看到评论。
- 点击删除按钮可以删除本地评论。

完成标准：

- 能根据路由参数中的影片 id 找到 mock 影片。
- 找不到影片时显示简单的未找到页面。
- 播放按钮能跳转到播放页。
- 收藏、评分和评论保存到 `localStorage`。
- 刷新页面后，本地收藏状态、评分和评论仍然存在。

### 播放页

用途：

- 播放影片。
- 第一阶段只使用 HTML5 `video`。
- 第一阶段支持本地弹幕最小版本。

基础内容：

- 视频播放器
- 影片标题
- 弹幕输入框和发送按钮
- 视频画面上方的弹幕覆盖层
- 返回详情页按钮
- 简单的影片信息

点击行为：

- 播放、暂停、全屏等使用浏览器原生控件。
- 发送弹幕时记录当前视频播放时间。
- 视频播放到对应时间点附近时显示本地弹幕。
- 返回详情页。

完成标准：

- 能读取影片 `playUrl`。
- 能正常渲染 HTML5 `video`。
- 测试视频能播放。
- 弹幕保存到 `localStorage`。
- 弹幕覆盖层不影响播放器控制按钮。

### 用户中心

用途：

- 展示用户侧本地数据。
- 第一阶段不做真实登录，只展示本地观看历史和本地收藏。

基础内容：

- 观看历史区域
- 我的收藏区域
- 空状态提示

完成标准：

- 页面能打开。
- 能从 `localStorage` 读取观看历史。
- 能从 `localStorage` 读取收藏列表。
- 没有观看历史或收藏时有简单提示。

## 3. 建议路由

| 页面 | 路由示例 | 说明 |
| --- | --- | --- |
| 首页 | `/` | 展示推荐和影片分区 |
| 分类页 | `/category/:category` | 例如 `/category/movie` |
| 搜索页 | `/search` | 第一阶段实现本地搜索 |
| 详情页 | `/video/:id` | 根据 id 展示影片详情 |
| 播放页 | `/watch/:id` | 根据 id 播放影片 |
| 用户中心 | `/profile` | 第一阶段展示本地历史和收藏 |

如果第一阶段不使用前端路由库，也可以用简单的 hash 路由，例如 `#/video/1`。

## 4. 建议组件拆分

组件拆分不要过度复杂，第一阶段建议只拆常用组件：

- `Navbar`：顶部导航
- `VideoCard`：影片卡片
- `VideoRow`：影片横向列表
- `HeroBanner`：首页主推荐
- `VideoGrid`：影片网格
- `PageLayout`：页面基础布局，可选

初学者可以先理解一个原则：页面负责组织数据和布局，组件负责复用一小块 UI。

## 5. 第一阶段 mock 数据文件

建议创建：

- `mocks/videos.ts`

数据示例结构：

```ts
export const videos = [
  {
    id: "video-001",
    title: "示例电影",
    category: "movie",
    coverUrl: "/images/demo-cover.jpg",
    bannerUrl: "/images/demo-banner.jpg",
    description: "这是一个用于前端展示的示例影片。",
    year: 2026,
    region: "美国",
    genres: ["科幻", "冒险"],
    duration: "1小时58分钟",
    playUrl: "https://example.com/demo.mp4"
  }
];
```

第一阶段不要追求数据字段完整，先保证页面够用。

## 6. 第一阶段已包含的本地交互

第一阶段已经包含以下本地交互：

- 搜索页根据关键词过滤影片。
- 分类页根据分类、地区、年份过滤影片。
- 详情页支持收藏按钮。
- 详情页支持本地评分。
- 详情页支持本地评论和删除评论。
- 播放页记录观看历史。
- 播放页支持本地弹幕。
- 用户中心展示收藏和观看历史。

数据暂时保存在 `localStorage`，不连接后端。

第一阶段本地交互完成标准：

- 搜索和分类筛选可用。
- 收藏刷新后不丢失。
- 播放记录刷新后不丢失。
- 用户中心能读取本地收藏和观看历史。
- 评分、评论、弹幕刷新后不丢失。
- 不接真实后端、不接数据库、不实现真实登录。
