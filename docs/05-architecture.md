# 架构设计初稿

## 1. 架构原则

项目按阶段推进，不一开始做完整大平台。

核心原则：

- 第一阶段只做前端 MVP 用户侧闭环，使用 mock 数据和 `localStorage`。
- 第二阶段只做前端体验整理和接口接入准备。
- 第三阶段再接入真实后端和数据库。
- 第四阶段再做 mp4 上传。
- 第五阶段再做 FFmpeg、HLS 和转码任务。
- 第六阶段再把评论、评分、弹幕升级为真实后端能力。

## 2. 第一阶段架构

第一阶段只有前端。

```text
浏览器
  |
  | 读取页面和静态资源
  v
前端应用
  |
  | 读取 mock 数据
  v
mocks/videos.ts
  |
  | playUrl
  v
HTML5 video 播放测试视频

前端应用
  |
  | 保存本地用户侧数据
  v
localStorage
```

特点：

- 没有后端。
- 没有数据库。
- 没有真实登录。
- 数据写在 mock 文件中。
- 播放地址写在影片数据的 `playUrl` 字段中。
- 搜索和分类筛选在浏览器中完成。
- 观看历史、收藏、评分、评论和弹幕保存到 `localStorage`。

第一阶段完成标准：

- 影片数据能从 mock 文件展示到页面。
- 用户能从首页或列表进入详情页。
- 用户能从详情页进入播放页。
- 播放页能播放测试视频。
- 用户能在本地使用观看历史、收藏、评分、评论和弹幕最小版本。

## 3. 第二阶段架构

第二阶段仍然只有前端，重点是整理第一阶段体验和为后端接口切换做准备。

```text
浏览器
  |
  v
前端应用
  |
  | 读取 mock 数据
  v
mocks/videos.ts
```

第二阶段关注点：

- 文档阶段边界一致。
- 页面组件职责清楚。
- mock 数据和类型定义一致。
- `localStorage` 工具函数清楚。
- 后续接 API 时能逐步替换 mock 数据来源。

第二阶段完成标准：

- 第一阶段用户侧功能保持可用。
- 前端代码和文档没有明显冲突。
- 不引入真实后端、数据库或复杂依赖。

## 4. 第三阶段架构

第三阶段引入真实后端和数据库。

```text
浏览器前端
  |
  | HTTP API
  v
后端服务
  |
  v
数据库
```

后端负责：

- 用户注册登录
- 影片列表和详情
- 分类列表
- 收藏
- 观看历史
- 后台影片新增、编辑、上下架

第三阶段仍然不负责：

- 视频上传
- 视频转码
- HLS
- 评论、评分、弹幕的真实后端能力

视频播放方式：

- 管理员手动填写 `playUrl`。
- 前端播放页读取后端返回的 `playUrl`。
- HTML5 `video` 直接播放该地址。

第三阶段完成标准：

- 前端页面能从后端接口读取影片。
- 管理员能维护影片基础信息。
- 普通用户能收藏和查看观看历史。
- 播放页能播放管理员填写的 mp4 地址。

## 5. 第四阶段架构

第四阶段引入后台 mp4 上传。

```text
管理员后台
  |
  | 上传 mp4
  v
后端服务
  |
  | 保存文件
  v
文件存储
  |
  | 返回文件地址
  v
影片 playUrl
```

可以先选择简单存储方式：

- 存到服务器本地目录。
- 或存到对象存储。

初期为了降低难度，可以先本地文件存储。后续正式部署时再迁移到对象存储。

第四阶段完成标准：

- 管理员可以上传 mp4。
- 后端保存文件并生成访问地址。
- 影片 `playUrl` 使用上传后的 mp4 地址。
- 前台播放页可以播放上传的视频。

## 6. 第五阶段架构

第五阶段引入 FFmpeg 和 HLS。

```text
管理员上传视频
  |
  v
后端创建转码任务
  |
  v
FFmpeg 转码
  |
  v
生成 HLS 文件
  |
  v
前端播放 m3u8
```

新增能力：

- 转码任务状态
- 转码失败记录
- m3u8 播放
- 后台查看任务状态

第五阶段完成标准：

- 上传后创建转码任务。
- 后台能看到任务状态。
- 转码成功后前端播放 m3u8。
- 转码失败时能看到失败原因。

## 7. 第六阶段架构

第六阶段增加互动功能。

新增模块：

- 评论服务
- 评分服务
- 弹幕服务

前台能力升级：

- 本地评论升级为登录用户评论。
- 本地评分升级为登录用户评分和平均分。
- 本地弹幕升级为真实接口保存和读取。

后台新增：

- 删除评论
- 删除弹幕

第六阶段完成标准：

- 登录用户可以评论。
- 登录用户可以评分。
- 登录用户可以发送弹幕。
- 管理员可以删除评论和弹幕。

## 8. 部署建议

第一阶段和第二阶段：

- 只需要部署前端静态页面。
- 可以使用任意静态托管方式。
- 第一阶段的 localStorage 数据只保存在用户浏览器中，不需要服务器存储。

第三阶段：

- 前端和后端可以分开部署。
- 数据库建议单独部署。

第四阶段以后：

- 视频文件不建议长期放应用服务器。
- 正式部署建议使用对象存储和 CDN。

但这些部署优化不需要在第一阶段处理。
