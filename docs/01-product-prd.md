# 类 Netflix 视频网站产品需求文档初稿

## 1. 项目背景

本项目最终目标是实现一个面向国内用户的类 Netflix 视频网站。网站部署在海外服务器，用户通过 Web 端访问。内容以电影、电视剧、动漫等影片为主，影片由平台自己维护。

当前阶段不追求完整平台能力，而是按阶段逐步推进。第一阶段定义为“前端 MVP 用户侧闭环”，先用 mock 数据和 `localStorage` 跑通用户侧核心体验：浏览影片、搜索筛选、查看详情、播放测试视频，以及观看历史、收藏、评分、评论、弹幕的最小本地版本。

## 2. 产品目标

### 最终目标

实现一个可运营的视频网站，具备以下能力：

- 用户可以浏览影片、搜索影片、查看详情、播放影片。
- 用户可以收藏影片、查看观看历史。
- 管理员可以维护影片信息，控制影片上下架。
- 后续支持视频上传、转码、评论、评分和弹幕。

### 第一阶段目标

第一阶段实现前端 MVP 用户侧闭环：

- 首页
- 分类页
- 搜索页
- 影片详情页
- 播放页
- 用户中心
- 本地搜索
- 本地分类筛选
- localStorage 观看历史
- localStorage 收藏
- localStorage 评分
- localStorage 评论
- localStorage 弹幕

第一阶段的数据全部来自 mock 文件，例如 `mocks/videos.ts`。播放页使用 HTML5 `video` 播放公开测试视频地址或本地静态 mp4。

第一阶段不接真实后端，不接数据库，不实现真实登录。收藏、观看历史、评分、评论和弹幕都只是浏览器本地 `localStorage` 最小版本。

## 3. 用户角色

### 游客

第一阶段游客可以：

- 浏览首页影片列表。
- 使用分类页按分类、地区、年份筛选影片。
- 使用搜索页按关键词搜索影片。
- 查看影片详情。
- 进入播放页播放测试视频。
- 在本地保存观看历史。
- 在本地收藏影片。
- 在本地给影片评分。
- 在本地发表评论和删除评论。
- 在播放页发送和查看本地弹幕。

### 普通用户

第一阶段已经包含普通用户相关体验的本地最小版本，但不做真实登录：

- 收藏影片。
- 查看本地观看历史。
- 查看用户中心。

### 管理员

第三阶段开始引入管理员：

- 新增影片。
- 编辑影片。
- 控制影片上架、下架。
- 手动填写播放地址 `playUrl`。

第一阶段不做后台管理页面。

## 4. 阶段划分

### 第一阶段：前端 MVP 用户侧闭环

目标：

- 使用 mock 数据完成前台页面结构。
- 建立页面之间的路由跳转关系。
- 用 HTML5 `video` 播放测试视频。
- 在前端本地实现搜索过滤和分类筛选。
- 用 `localStorage` 实现观看历史、收藏、评分、评论和弹幕的最小版本。

不做：

- 真实后端
- 数据库
- 登录注册
- 视频上传
- 视频转码
- 评论、评分、弹幕的真实后端能力
- 真实收藏和真实历史接口
- 后台管理

完成标准：

- 首页能展示 mock 影片。
- 分类页能按 mock 数据筛选影片。
- 搜索页能基于 mock 数据搜索影片。
- 详情页能根据影片 id 展示影片信息。
- 播放页能播放测试视频。
- 用户中心能展示本地观看历史和本地收藏。
- 详情页可以使用本地收藏、评分和评论。
- 播放页可以使用本地弹幕。

### 第二阶段：前端体验整理和接口接入准备

目标：

- 在第一阶段基础上整理前端体验和代码质量。
- 补齐必要的空状态、错误提示和基础响应式细节。
- 为第三阶段从 mock 数据切换到真实接口做准备。

不做：

- 真实用户系统
- 真实后端接口
- 数据库
- 视频上传和转码

完成标准：

- 文档和代码阶段边界一致。
- 前端用户侧功能结构清晰。
- localStorage 数据读写逻辑清楚。
- 没有为了接后端提前引入复杂依赖。

### 第三阶段：真实后端基础接口

目标：

- 实现用户、影片、分类、收藏、观看历史的基础接口。
- 引入真实数据库。
- 后台管理只支持影片新增、编辑、上下架。
- 视频地址由管理员手动填写 `playUrl`。

不做：

- 视频文件上传
- FFmpeg 转码
- HLS
- 评论、评分、弹幕
- 复杂后台统计

完成标准：

- 前端影片列表来自后端接口。
- 影片详情来自后端接口。
- 登录用户可以收藏影片。
- 登录用户可以查看观看历史。
- 管理员可以新增、编辑、上架、下架影片。
- 管理员手动填写的视频地址可以在播放页播放。

### 第四阶段：后台 mp4 上传

目标：

- 后台支持上传 mp4 原文件。
- 上传完成后生成可访问播放地址。
- 前台播放页直接播放 mp4。

不做：

- HLS 转码
- 多清晰度
- 复杂上传断点续传
- 转码任务队列

完成标准：

- 管理员可以上传 mp4 文件。
- 上传成功后影片记录自动保存播放地址。
- 前台播放页可以播放该 mp4 文件。

### 第五阶段：FFmpeg 转码和 HLS

目标：

- 引入 FFmpeg。
- 上传视频后创建转码任务。
- 转码为 HLS 格式。
- 前端播放 m3u8。
- 后台展示任务状态。

完成标准：

- 上传视频后能看到任务状态：等待中、转码中、成功、失败。
- 转码成功后，播放页可以播放 m3u8 地址。
- 转码失败时，管理员能看到失败状态。

### 第六阶段：评论、评分和弹幕

目标：

- 将第一阶段的本地评论、评分、弹幕升级为真实后端能力。
- 登录用户可以评论影片。
- 登录用户可以给影片评分。
- 登录用户可以在播放页发送弹幕。
- 管理员可以删除不合适的评论和弹幕。

完成标准：

- 用户登录后可以提交评论。
- 用户登录后可以评分。
- 播放页可以展示和发送弹幕。
- 管理员可以删除评论和弹幕。

## 5. 第一阶段核心链路

第一阶段先验证一条主链路：

1. mock 文件中有影片数据。
2. 首页、分类页、搜索页读取并展示影片数据。
3. 用户点击某个影片卡片。
4. 跳转到影片详情页。
5. 用户点击播放按钮。
6. 跳转到播放页。
7. HTML5 `video` 播放 `playUrl`。

在这条链路清楚后，第一阶段允许继续在前端本地扩展观看历史、收藏、评分、评论和弹幕。它们都只保存在 `localStorage`，不代表真实用户数据或真实后端能力。

## 6. 第一阶段影片 mock 数据建议

每个影片至少包含：

- `id`：影片唯一标识
- `title`：影片标题
- `category`：分类，例如 movie、tv、anime
- `coverUrl`：封面图
- `bannerUrl`：详情页横幅图，可选
- `description`：简介
- `year`：年份
- `region`：地区
- `genres`：题材标签
- `duration`：时长
- `playUrl`：测试视频地址

电视剧和动漫可以先简单处理成单个影片，不急着做多集管理。多集能力可以在后续阶段再扩展。

## 7. 暂不纳入 MVP 的能力

以下能力不要在第一阶段实现：

- 真实后台管理
- 真实用户登录
- 数据库设计落地
- 文件上传
- 视频转码
- 多清晰度播放
- 付费会员
- DRM
- 推荐算法
- 评论、评分、弹幕的真实后端接口
- 评论、评分、弹幕的后台审核管理
- 后台统计
