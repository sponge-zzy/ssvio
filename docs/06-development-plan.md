# 阶段化开发计划初稿

## 1. 开发原则

本项目按六个阶段推进。每个阶段只完成当前阶段的目标，不提前实现后面阶段的复杂功能。

第一阶段的重点不是做完整平台，而是让前端初学者理解：

- 页面怎么拆。
- 路由怎么跳。
- mock 数据怎么展示。
- 详情页和播放页怎么通过 id 找到同一部影片。
- `localStorage` 怎么承载最小用户侧互动。

## 2. 第一阶段：前端 MVP 用户侧闭环

目标：

- 实现首页、分类页、搜索页、详情页、播放页、用户中心。
- 使用 `mocks/videos.ts` 提供影片数据。
- 播放页使用 HTML5 `video` 播放测试视频。
- 在前端本地实现搜索和分类筛选。
- 使用 `localStorage` 实现观看历史、收藏、评分、评论和弹幕的最小版本。
- 不接真实后端，不接数据库，不实现真实登录。

建议开发顺序：

1. 建立基础页面和路由。
2. 编写 mock 影片数据。
3. 实现通用组件：导航栏、影片卡片、影片列表。
4. 实现首页。
5. 实现分类页筛选。
6. 实现搜索页本地搜索。
7. 实现详情页，通过 id 查找影片。
8. 实现播放页，通过 id 读取 `playUrl`。
9. 播放页进入时记录观看历史。
10. 详情页增加收藏、评分和评论的 `localStorage` 最小版本。
11. 播放页增加弹幕的 `localStorage` 最小版本。
12. 用户中心读取观看历史和收藏。
13. 检查所有页面跳转和本地数据刷新后是否保留。

完成标准：

- 所有第一阶段页面都能打开。
- 首页能展示 mock 影片。
- 影片卡片能跳转详情页。
- 详情页播放按钮能跳转播放页。
- 播放页能播放测试视频。
- 搜索和分类筛选可用。
- 观看历史和收藏刷新后不丢失。
- 评分、评论和弹幕刷新后不丢失。
- 用户中心能展示本地观看历史和本地收藏。

## 3. 第二阶段：前端体验整理和接口接入准备

目标：

- 整理第一阶段代码和文档边界。
- 优化空状态、错误提示和基础响应式体验。
- 检查 mock 数据、类型定义、路由和 `localStorage` 工具函数。
- 为第三阶段切换真实接口做准备，但不接真实后端。

建议开发顺序：

1. 检查文档和代码是否一致。
2. 检查页面职责是否清楚。
3. 检查公共组件是否存在重复代码。
4. 检查 `localStorage` 工具函数是否清楚。
5. 检查类型定义和 mock 数据字段是否一致。
6. 检查路由跳转是否正常。
7. 检查 CSS 是否影响播放器和弹幕操作。

完成标准：

- 文档阶段边界清楚。
- 前端业务代码没有无关功能或明显重复。
- 本地用户侧功能仍然可用。
- 构建通过。

## 4. 第三阶段：真实后端基础接口

目标：

- 接入真实后端。
- 接入数据库。
- 实现用户、影片、分类、收藏、观看历史。
- 后台支持影片新增、编辑、上架、下架。
- 管理员手动填写 `playUrl`。

建议开发顺序：

1. 确定后端技术栈和数据库。
2. 建立用户表、分类表、影片表、收藏表、观看历史表。
3. 实现注册、登录、获取当前用户。
4. 实现影片列表和影片详情接口。
5. 实现分类列表接口。
6. 实现收藏接口。
7. 实现观看历史接口。
8. 实现后台影片新增、编辑、上架、下架接口。
9. 前端从 mock 数据切换到真实接口。
10. 保留 mock 数据作为开发备用。

完成标准：

- 前端影片数据来自后端。
- 用户可以注册登录。
- 登录用户可以收藏影片。
- 登录用户可以查看观看历史。
- 管理员可以维护影片。
- 管理员填写的 `playUrl` 可以被播放页播放。

## 5. 第四阶段：后台 mp4 上传

目标：

- 管理员可以上传 mp4 文件。
- 上传后生成可访问播放地址。
- 播放页直接播放 mp4。

建议开发顺序：

1. 后台新增上传入口。
2. 后端实现 mp4 文件接收。
3. 保存文件到指定目录或对象存储。
4. 返回文件访问地址。
5. 保存到影片的 `playUrl`。
6. 前端播放页测试播放上传文件。

完成标准：

- 管理员能上传 mp4。
- 上传成功后影片自动拥有播放地址。
- 前端能播放上传后的 mp4。

## 6. 第五阶段：FFmpeg 转码和 HLS

目标：

- 引入 FFmpeg。
- 上传视频后创建转码任务。
- 转码生成 HLS 文件。
- 前端播放 m3u8。
- 后台显示任务状态。

建议开发顺序：

1. 安装和验证 FFmpeg。
2. 设计转码任务表。
3. 上传视频后创建任务。
4. 后端执行转码命令。
5. 保存 m3u8 和 ts 文件。
6. 更新任务状态。
7. 播放页支持 m3u8 播放。
8. 后台展示任务状态和失败原因。

完成标准：

- 上传后自动进入转码任务。
- 任务状态能正确变化。
- 转码成功后播放页播放 m3u8。
- 转码失败时后台能看到失败原因。

## 7. 第六阶段：评论、评分和弹幕

目标：

- 将第一阶段的本地评论、评分、弹幕升级为真实后端能力。
- 登录用户评论。
- 登录用户评分。
- 播放页真实弹幕数据。
- 管理员删除评论和弹幕。

建议开发顺序：

1. 实现评论表和评论接口。
2. 详情页展示评论列表。
3. 登录用户提交评论。
4. 实现评分表和评分接口。
5. 详情页展示平均分。
6. 实现弹幕表和弹幕接口。
7. 播放页按时间展示弹幕。
8. 后台支持删除评论和弹幕。

完成标准：

- 登录用户可以评论。
- 登录用户可以评分。
- 播放页可以发送和展示弹幕。
- 管理员可以删除评论和弹幕。

## 8. 每阶段交付物

| 阶段 | 交付物 |
| --- | --- |
| 第一阶段 | 前端 MVP 用户侧闭环、mock 数据、HTML5 video 播放、localStorage 历史/收藏/评分/评论/弹幕 |
| 第二阶段 | 前端体验整理、代码质量检查、接口接入准备 |
| 第三阶段 | 后端基础接口、数据库、简单后台影片管理 |
| 第四阶段 | mp4 上传和播放 |
| 第五阶段 | FFmpeg 转码、HLS、任务状态 |
| 第六阶段 | 评论、评分、弹幕的真实后端能力和后台审核 |

## 9. 初学者注意事项

- 不要一开始就做后台。
- 不要一开始就做数据库。
- 不要一开始就研究转码。
- 先让页面能跑起来。
- 先让 mock 影片能展示和播放。
- 后续每个功能都建立在这条主链路上。
