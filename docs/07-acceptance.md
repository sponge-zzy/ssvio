# 验收标准初稿

## 1. 验收说明

本文件用于判断每个阶段是否真正完成。每个阶段都应该先通过验收，再进入下一阶段。

验收重点不是代码写了多少，而是用户能不能完成当前阶段定义的核心动作。

## 2. 第一阶段验收：前端 MVP 用户侧闭环

阶段目标：

- 使用 mock 数据实现前端用户侧页面。
- 跑通“列表展示、详情查看、播放视频”主链路。
- 基于 mock 数据实现搜索和筛选。
- 使用 `localStorage` 实现观看历史、收藏、评分、评论和弹幕的最小本地版本。
- 不接真实后端，不接数据库，不实现真实登录。

验收项：

| 编号 | 验收项 | 是否必须 |
| --- | --- | --- |
| 1 | 首页可以打开 | 必须 |
| 2 | 首页能展示 mock 影片 | 必须 |
| 3 | 分类页可以打开 | 必须 |
| 4 | 搜索页可以打开并基于 mock 数据搜索 | 必须 |
| 5 | 详情页能根据影片 id 展示影片信息 | 必须 |
| 6 | 找不到影片时有简单提示 | 必须 |
| 7 | 播放页能根据影片 id 读取 `playUrl` | 必须 |
| 8 | HTML5 `video` 能播放测试视频 | 必须 |
| 9 | 分类页可以按分类、地区、年份筛选 | 必须 |
| 10 | 页面导航和跳转清晰 | 必须 |
| 11 | 进入播放页后会生成本地观看历史 | 必须 |
| 12 | 详情页可以收藏和取消收藏影片 | 必须 |
| 13 | 详情页可以保存本地评分 | 必须 |
| 14 | 详情页可以添加和删除本地评论 | 必须 |
| 15 | 播放页可以发送并按时间展示本地弹幕 | 必须 |
| 16 | 用户中心可以展示本地观看历史和本地收藏 | 必须 |

第一阶段完成定义：

- 从首页点击一部影片，能进入详情页。
- 从详情页点击播放，能进入播放页。
- 播放页能播放测试视频。
- 搜索、筛选、观看历史、收藏、评分、评论、弹幕都可以在本地完成。
- 全过程不需要后端、数据库和真实登录。

## 3. 第二阶段验收：前端体验整理和接口接入准备

阶段目标：

- 整理第一阶段代码和文档。
- 保持用户侧功能可用。
- 为第三阶段接真实接口做准备，但不接真实后端。

验收项：

| 编号 | 验收项 | 是否必须 |
| --- | --- | --- |
| 1 | 文档阶段边界和当前代码一致 | 必须 |
| 2 | 页面组件职责清楚 | 必须 |
| 3 | 公共组件职责清楚 | 必须 |
| 4 | mock 数据字段和类型定义一致 | 必须 |
| 5 | localStorage 读写逻辑清楚 | 必须 |
| 6 | 路由跳转清晰 | 必须 |
| 7 | CSS 不影响播放器控制按钮 | 必须 |
| 8 | 构建通过 | 必须 |

第二阶段完成定义：

- 第一阶段用户侧功能保持可用。
- 文档、类型、mock 数据、路由和 CSS 没有明显冲突。
- 没有新增后端、数据库或复杂依赖。

| 6 | 登录用户可以收藏影片 | 必须 |

## 5. 第四阶段验收：后台 mp4 上传

阶段目标：

- 管理员上传 mp4 原文件。
- 前台直接播放上传后的 mp4。

验收项：

| 编号 | 验收项 | 是否必须 |
| --- | --- | --- |
| 1 | 后台有 mp4 上传入口 | 必须 |
| 2 | 后端能接收 mp4 文件 | 必须 |
| 3 | 上传成功后能生成访问地址 | 必须 |
| 4 | 访问地址能保存到影片 `playUrl` | 必须 |
| 5 | 播放页能播放上传后的 mp4 | 必须 |

第四阶段完成定义：

- 影片可以通过后台上传文件进入系统，并在前台播放。

## 8. 不通过验收的常见情况

第一阶段常见问题：

- 页面看起来有了，但影片卡片不能跳转。
- 详情页不是根据 id 展示数据。
- 播放页没有真正读取 `playUrl`。
- 视频播放器只是占位，不能播放测试视频。
- 搜索、筛选、收藏、历史、评分、评论、弹幕只在内存里有效，刷新后丢失。
- 把 localStorage 最小版本写成了真实登录、真实接口或后台审核能力。

第二阶段常见问题：

- 文档仍然把已完成的本地互动写到后续阶段。
- 为了整理代码引入新依赖或新增业务功能。
- 修改 CSS 后影响播放器控制按钮。

第三阶段常见问题：

- 前端仍然依赖 mock 数据。
- 后台新增影片后，前台看不到。
- 下架影片后，普通用户仍然能看到。

第四阶段常见问题：

- 文件上传了，但没有保存到影片播放地址。
- 上传后的地址浏览器无法访问。

第五阶段常见问题：

- 转码失败没有记录原因。
- m3u8 地址生成了，但前端不能播放。

第六阶段常见问题：

- 评论和弹幕没有管理员删除能力。
- 评分没有限制同一用户重复提交。
